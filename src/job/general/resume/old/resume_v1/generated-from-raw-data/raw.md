## Arbor The Tree: 진도 및 오답 관리 시스템

### 소스 코드 링크

- https://github.com/ThePott/arbor-the-tree

### 해결하려고 한 문제

#### 기억에 의존해야 하는 진도 추적

- 입시 수학 학원 강사는 진도를 매번 기억에 의존
- 개별 수업의 경우, 한 수업에 10~20명의 학생을 가르칠 때 모두의 진도, 숙제를 기억해야 함

#### 학생에게만 맡기게 되는 오답 관리

- 틀린 문제를 다시 풀었는지 확인할 방법이 없음
- 한 문제집을 여러번 풀면 좋다고 하지만 수업으로 강제하기가 불가능

#### 기존 파이썬 시스템의 문제점 해결

- 강사 재직 시절, 구글 시트와 파이썬을 이용해 비슷한 시스템을 구축함
- 진도표와 오답 과제 현황을 수기로 동기화해야 했음(학생 당 5분, 수업당 50분에서 100분 소요)

### 기능과 그 기능이 해결한 문제

#### 칸반 보드의 진도 관리

- 해결한 문제: 오답 체크, 혹은 과제의 상황에 맞게 진도표 자동 갱신
- 문제집별, 혹은 진행 상황별("전 숙제", "오늘", "새 숙제")로 진도를 관리할 수 있는 칸반 보드
- 한 섹션의 오답 체크를 다 하면 자동으로 완료 표시
- 오답 과제가 추가되면 자동으로 숙제로 기록

#### 그리드 체크박스로 오답 체크

- 해결한 문제: 오답 체크를 학생에게 맡겨 오답 체크를 db에 입력하는 시간 단축
- 다중 체크와 개별 체크를 통해 학생이 빠르게 당일 수업의 정오를 기록할 수 있게 함

#### 오답을 모아 학습지로 제작, 숙제로 배부

- 해결한 문제: 데이터베이스를 구글 시트에서 PostgreSQL로 바꿔 학습지 제작 속도를 10초에서 0.3초로 단축 (33배 향상)

## Harpooning Tmux: 생산성 향상 툴

### 소스 코드 링크

- https://github.com/ThePott/harpooning-tmux

### 해결하려고 한 문제

- tmux로 여러 프로젝트를 왔다갔다 하는데 tmux의 세션은 순서를 바꿀 수가 없음
- tmux 서버를 새로 켤 때마다 필요한 프로젝트 경로로 이동하고 tmux에 새 세션으로 추가하기까지 시간이 걸림

### 기능과 그 기능이 해결한 문제

- `fzf`를 이용해 `.git/`이 있는 디렉토리만 검색 -> 선택하면 새 세션으로 추가
- 세션 목록을 팝업창으로 띄워 관리: 목록의 순서대로 숫자 단축키로 이동 가능
- 세션 목록을 팝업창으로 띄워 관리: 목록과 실제 세션 목록 동기화. 목록에서 삭제하면 실제 세션에서도 삭제됨

### 성과

- 새 프로젝트를 tmux 세션으로 등록하는 시간이 20초에서 2초로 단축됨 (10배 향상)
