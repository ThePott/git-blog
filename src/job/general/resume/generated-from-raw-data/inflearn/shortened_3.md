# 질문 03. 끝이 좋았던 경험과 좋지 못했던 경험

## 끝이 좋았던 경험

학원 강사 생활 마지막 날, 학생 두 명이 감사 인사를 보내왔습니다. 오랜 시간이 지나 다시 연락한 학생은 "수업이 종종
생각나고, 덕분에 더 잘하게 되었다"고 했습니다. 힘든 강사 생활을 그렇게 보답받았고, 이후에도 누군가의 성장에 함께하고
싶다고 생각했습니다. "성장기회의 평등"을 위해 앞장서는 인프런과 함께 그들의 성장에 기여하고 싶습니다.

## 끝이 좋지 못했던 경험

부트캠프 심화반에서 3개월간 Arbor The Tree 프로토타입을 개발했습니다. 마지막 발표 당일, 수업 직전에야 배포했는데 발표
시점에 배포 오류를 발견했습니다. 다른 수강생들이 모두 완성했을 때 저만 미완성이었고, 남들이 이용할 수도 없었습니다.

### 문제점

1. 개발 서버에서만 작업하다 배포 직전에 빌드하니 처음 보는 오류들이 발생
2. DB 변경(MongoDB → PostgreSQL) 후 API URL을 일괄 교체하지 못함
3. 배포 환경에서만 발생하는 오류 해결 경험 부족

### 해결책 적용

- API URL을 환경변수로 관리하여 로컬/배포 서버 간 쉽게 전환
- husky pre-push 훅에 build 명령어 추가하여 수시로 빌드 오류 확인
- 배포는 최소 하루 전에 진행, 오류 해결 시간 확보

Arbor The Tree 실제 개발 시 고객과 2주 간격 미팅마다 전날 배포, 당일 테스트하는 루틴을 정착시켜 모든 미팅에서 진척
상황을 성공적으로 전달했습니다.

### 교훈: Fail Fast

개발하며 "Fail Fast"를 되새깁니다. 최대한 빨리 실패하고 수정하는 루틴을 정착시켰습니다. 의도되지 않은 상황에서는 에러를
던지는 등, 실패를 빨리 하고 확실하게 확인한다는 원칙을 적용하며 작업하고 있습니다.
