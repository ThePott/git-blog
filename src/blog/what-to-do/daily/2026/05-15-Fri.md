`./src/features/_sidebar._assigned._check.check/types/index.ts`에서 오답 체크 변동을 추적하기 위해
`ReviewCheckChangedInfo` 타입을 만들 던 중 있던 일입니다. 오답 체크를 할 때 문제집의 경우에만 `session_id`를 body에 담아
요청해야 하고, 오답과제를 체크할 땐 그럴 필요가 없었습니다. 이는 오답과제와 문제집의 스키마가 다름에서 발생하는
문제였습니다.

`session_id`를 optional로 하면 session_id를 어느 경우 작성해야 하는지가 명확하지 않게 되고, 두 개의 타입으로 나누자니
거의 같은 코드를 반복하게 되었습니다. 이를 해결하려 Discriminated Union을 사용해 optional 없이 타입을 만들었습니다.
이로써 `forWhat` 프로퍼티의 값에 따라 나머지 프로퍼티들의 타입을 명확히 구분할 수 있었습니다.

```ts
export type ReviewCheckChangedInfo =
    | {
          forWhat: "session"
          status: AttemptStatus | null
          indexInfo: IndexInfo
          session_id: string | null // NOTE: null이면 오늘, 혹은 숙제로 할당 안 받은 문제, 따라서 체크 비활성화
      }
    | {
          forWhat: "assignment"
          status: AttemptStatus | null
          indexInfo: IndexInfo
      }
```
