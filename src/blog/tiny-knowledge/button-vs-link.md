## Button vs Link

|               | **Button (`<button>`)**  | **Link (`<a>`)**              |
| ------------- | ------------------------ | ----------------------------- |
| **목적**      | 액션 실행                | 페이지 이동                   |
| **예시**      | 폼 제출, 모달 열기, 삭제 | 다른 페이지로 이동, 앵커 이동 |
| **키보드**    | `Space` + `Enter`        | `Enter`만                     |
| **기본 동작** | 없음                     | URL로 이동                    |
| **SEO**       | 영향 없음                | 크롤러가 따라감               |
| **우클릭**    | 의미 없음                | "새 탭에서 열기" 가능         |

## 핵심 원칙

```
클릭하면 URL이 바뀌나? → <a>
클릭하면 동작이 실행되나? → <button>
```

## 흔한 실수

```tsx
// ❌ 잘못됨 - 버튼처럼 생겼지만 링크임
<a href="#" onClick={handleSubmit}>제출</a>
// ❌ 잘못됨 - 링크처럼 생겼지만 버튼임
<button onClick={() => navigate('/home')}>홈으로</button>
// ✅ 올바름
<button onClick={handleSubmit}>제출</button>
<a href="/home">홈으로</a>
```

## 접근성 관점

- 스크린 리더는 button과 link를 다르게 읽음
- 잘못 사용하면 사용자가 혼란스러워함
- `<div onClick>` 최악 → 키보드 접근 불가
