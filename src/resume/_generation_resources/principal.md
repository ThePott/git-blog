# 이력서 생성 원칙

## 입력

- `raw.md`: 지원자의 모든 정보
- `key-message.md`: 핵심 메시지
- 채용 공고 + 키워드 (회사별로 다름)

## XYZ Formula

모든 성과는 이 구조로:

```
Accomplished [X] as measured by [Y] by doing [Z]
```

- X: 성과/결과
- Y: 측정 지표 (%, 시간, 금액 등)
- Z: 사용한 방법/기술

나쁜 예: "소셜 미디어 계정 관리" 좋은 예: "타겟 콘텐츠 캘린더 도입(Z)으로 팔로워 30%, 트래픽 15% 증가(X, Y)"

## 구조: 트리

```
전체 키 메시지 (나는 어떤 개발자인가)
└── 프로젝트 키 메시지 (이 프로젝트로 무엇을 증명하나)
    └── 페이지/섹션 키 메시지 (이 내용이 프로젝트를 어떻게 뒷받침하나)
```

모든 내용은 상위 메시지를 뒷받침해야 함.

## 문제 정의

- 고객의 요구에서 시작
- 팩트와 데이터로 출발
- 부모 문제 → 자식 문제로 흐르는 flow

## 자기소개 (신입)

한 페이지, 아래 순서:

1. 지원 동기 (30%)
2. 직무 역량
3. 문제 해결 능력: why → approach → solution → result

## 글쓰기 원칙

- 해석하게 쓰지 말고 결론을 써라
- one slide one message (양 많으면 분할)
- 이미지엔 어디에 집중해야 하는지 표시

## 주의사항

- 없는 사실 만들지 말 것
- 필요한 데이터 있으면 질문할 것
- 휴먼 피드백 필수: 키 메시지가 전달되었는지 확인
