# 프로젝트: Arbor The Tree

## 개요

- 진도 및 오답 관리 시스템
- https://github.com/ThePott/arbor-the-tree

## 해결하려고 한 문제

### 입시 학원 시스템의 문제

- 진도는 강사의 기억에 의존
- 오답 복습은 학생에게 위임 후 관리 안 됨
- 같은 문제집 반복 숙달이 좋다는 걸 알지만 추적이 안 되어 새 문제집만 계속 풀림

### 기존 파이썬 시스템의 문제

- 구글 시트를 DB로 사용 → 느림
- CLI로만 작동 → 다른 강사가 사용법 습득 불가
- 병렬적 자료 생성, 직렬적 동기화 → 병목 현상
    - 학생당 5분, 수업당 50~100분 소요

### 고객(원장) 요구사항

- 빠른 반응속도
- 쉬운 조작법 (아르바이트생 잦은 교체에도 인수인계 원활해야)

## 핵심 기능

### 1. 칸반 보드 진도 관리

- 문제집별, 진행 상황별("전 숙제", "오늘", "새 숙제")로 진도 관리
- 오답 체크 완료 시 자동으로 완료 표시
- 오답 과제 추가 시 자동으로 숙제로 기록

### 2. 그리드 체크박스 오답 체크

- 학생이 직접 당일 수업의 정오 기록
- 다중 체크, 개별 체크 지원
- 강사가 하던 작업을 학생에게 위임 → 시간 단축

### 3. Grid Input (문제집 등록)

- 구글 시트의 장점(자동완성, formula)은 차용
- 구글 시트의 단점(단원명 일일이 입력, 유효성 검사 어려움)은 개선
- 자동완성 + 내장 유효성 검사

### 4. Server-side PDF 생성 (Typst)

- 오답만 추출하여 학습지 PDF 생성
- 10~20명 학생의 오답과제를 한 번에 생성해야 함 → 빠른 생성 시간 필수

#### 기술 선택 과정

| 후보               | 장단점                                                        | 선택 |
| ------------------ | ------------------------------------------------------------- | ---- |
| React-PDF (client) | 브라우저 메인 스레드 점유, 30쪽 이상 Web Worker 권장          | X    |
| Puppeteer (server) | 브라우저 가상화 도구에 PDF 생성이 곁가지 기능, 오버엔지니어링 | X    |
| Typst (server)     | LaTex보다 빠름, 문법 간결, 브라우저와 무관                    | O    |

#### 문제의 본질

- PDF는 브라우저와 무관 → 서버에서 생성

#### 배포 이슈 해결

- Typst 컴파일러: npm 미지원 → bash 빌드 스크립트로 curl 설치
- 한글 폰트: typst root 내부에 위치해야 → 빌드 후 dist 폴더로 이동

#### 성과 (A/B 테스트: Typst vs React-PDF)

| 페이지 수 | 생성 시간 단축 |
| --------- | -------------- |
| 8쪽       | 4.7%           |
| 80쪽      | 30.7%          |
| 800쪽     | 61.3%          |

### 5. Zustand Local Store (컴포넌트별 상태 관리)

- 문제: 복잡한 공통 컴포넌트에서 부모 → 자식 상태 전달이 번거로움
- Context API도 불편 (상태 하나 만들 때마다 boilerplate)
- 해결: Zustand로 컴포넌트별 local store 생성

### 6. 서버 응답 시간 단축

#### 문제

- 문제집 등록 시 7초 소요
- 연속 요청 시 4~5초로 줄어듦

#### 진단

- cold start: 최초 응답에서만 시간 오래 걸림
- far region: 한국 <-(ok)-> Railway(싱가폴) <-(to far)-> Neon(N. Virginia)
- Neon 무료 플랜: cold start 강제, region 변경 불가

#### 해결

- Neon 제거, Railway에 PostgreSQL 직접 배포
- API와 DB를 같은 internal server에 배포 → round-trip latency 최소화

#### 성과

- 문제집 생성 시간: 7초 → 315ms (22배 향상)

# 프로젝트: Harpooning Tmux

## 개요

- 생산성 향상 툴
- https://github.com/ThePott/harpooning-tmux

## 해결하려고 한 문제

- tmux 세션 순서 변경 불가
- 새 프로젝트 추가 시 경로 이동 + 세션 추가까지 시간 소요

## 기능

- `fzf`로 `.git/` 있는 디렉토리 검색 → 선택 시 새 세션으로 추가
- 팝업창에서 세션 목록 관리: 순서대로 숫자 단축키 이동
- 목록에서 삭제 시 실제 세션도 삭제

## 성과

- 새 프로젝트를 tmux 세션으로 등록하는 시간: 20초 → 2초 (10배 향상)
