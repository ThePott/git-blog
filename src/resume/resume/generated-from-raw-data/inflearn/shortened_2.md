입시 학원 강사로 4년 6개월 재직하며 학습 과정의 비효율을 직접 경험했습니다. 진도와 오답 관리를 수기로 동기화하는 데
학생당 5분, 수업당 50~100분이 소요되었고, 이 문제를 해결하고자 직접 웹앱(Arbor The Tree)을 제작했습니다.

인프런은 "성장기회의 평등"이라는 미션 아래 150만 명 이상의 사용자에게 학습 환경을 제공하고 있습니다. 저 역시 학습의
비효율을 기술로 해결한 경험이 있기에, 인프런의 미션에 깊이 공감합니다. 학습자의 입장에서 문제를 정의하고 기술로 해결책을
만들어본 경험이 인프런에서 가치를 만드는 데 기여할 수 있다고 생각합니다.

인프런을 이용하며 개선하고 싶은 부분들이 있었습니다.

- Layout Shift 해결
    - `scrollbar-gutter:stable`로 스크롤바 유무와 무관하게 컨텐츠 위치 유지. 현재 랜딩 페이지에서 스켈레톤은 스크롤바가
      없고, 로딩 완료 시 스크롤바가 생기며 컨텐츠가 밀림.
    - `<main/>`의 최소 높이를 설정하여 로딩 중에도 푸터가 뷰포트 아래에 위치하게 함. 현재는 로딩 전후로 푸터가 깜빡임.

- 필터 기능 일원화
    - 현재 검색창(상단)과 카테고리 선택(중단)이 분리되어 있고, 기능적으로도 연동되지 않음. 예: "데이터 사이언스"
      카테고리에서 "초급" 검색 불가.
    - 검색창과 카테고리 선택을 통합하고 연동하면 사용자 경험 개선 가능.

- 사이드바 목차로 데스크탑 UI 최적화
    - 긴 글의 헤딩을 사이드바에 표시하여 글의 흐름을 쉽게 파악하게 함.
    - 클릭 시 해당 섹션으로 자동 스크롤.

- 무한 스크롤로 체류 시간 증대
    - 가상 스크롤로 돔 노드 관리하여 성능 최적화.
    - 푸터는 "이용 약관", "개인 정보 처리 방침" 등 법적 필수 항목만 최소화하여 항상 노출, 펼치기 버튼으로 전체 푸터
      표시.

인프런 사용자의 입장에서 문제를 찾고, 만족할만한 해결책을 만들겠습니다.
